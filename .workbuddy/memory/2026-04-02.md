# 2026-04-02 工作日志

## 项目概况
- 项目：狗狗数据分析系统（Flask + MySQL + PyEcharts）
- 技术栈：Flask 3.1.3, Flask-Login, Flask-SQLAlchemy, Flask-WTF, APScheduler, PyEcharts, Pandas
- 请求：团队技术能力提升，资深开发者代码审查与指导

## 代码审查发现的主要问题
1. charts.py 使用裸 pymysql 连接，而非 SQLAlchemy ORM，连接未使用连接池/上下文管理器
2. errors.py 存在 Bug：`request` 对象在函数外部导入（放在文件末尾），400错误处理器引用了未导入的 `request`
3. utils/api_response.py 中 `time` 模块在文件末尾导入（import 顺序不规范）
4. app.py 路由全部定义在一个函数内（未使用 Blueprint），随项目增长将变得难以维护
5. api/routes.py 获取 current_user_id 的方式有 Bug：`current_app.config.get('current_user_id', None)` 不正确
6. charts.py SQL 存在 SQL 注入风险（get_dog_food_list 使用 f-string 拼接 limit）
7. requirements.txt 存在重复依赖项、注释格式不规范
8. conftest.py logged_in_client fixture 密码写错（123 而非 123456）
