## 🔴 Git版本管理

### ❌ 错误1: 不检查版本号直接提交

**问题描述**:  
提交代码时使用错误的版本号(如v4.9.5),但Git历史中已有v4.9.13,导致版本号混乱。

**正确做法**:
```bash
# 1. 提交前先查看最新版本号
git log --oneline -1

# 2. 根据语义化版本规则确定新版本号
# - PATCH修复: v4.9.13 → v4.9.14
# - MINOR功能: v4.9.13 → v4.10.0
# - MAJOR重大: v4.9.13 → v5.0.0

# 3. 提交代码
git add .
git commit -m "v4.9.14"

# 4. 更新CHANGELOG,确保与Git版本号一致
```

**预防措施**:
- ✅ 每次提交前必须执行 `git log --oneline -1`
- ✅ 参考CHANGELOG中的最新版本号
- ✅ 遵循语义化版本规范(MAJOR.MINOR.PATCH)

---

### ❌ 错误2: 修改文件后不立即提交

**问题描述**:  
完成代码修改后,没有立即提交到Git,违反了"代码必须用Git管理"的规范。

**正确做法**:
```bash
# 每次修改完成后立即提交
git add <修改的文件>
git commit -m "vX.Y.Z - 简短描述"
```

**预防措施**:
- ✅ 养成"修改即提交"的习惯
- ✅ 不要累积多个修改再一次性提交
- ✅ 每次提交都要有清晰的commit message

## 📝 代码规范

### ❌ 错误3: 未遵循TDD流程

**问题描述**:  
先写实现代码,后写测试,违反了测试驱动开发(TDD)原则。

**正确流程**:
```
🔴 Red: 先写测试用例(预期失败)
  ↓
🟢 Green: 实现功能代码(使测试通过)
  ↓
🔵 Refactor: 重构优化(保持测试通过)
```

**预防措施**:
- ✅ 任何新功能必须先写测试
- ✅ 测试覆盖率 ≥ 80%
- ✅ 测试失败不得提交代码

---

### ❌ 错误4: 代码中有重复逻辑

**问题描述**:  
在`handle_breed_info()`函数中,上下文处理逻辑重复出现两次。

**正确做法**:
- 删除重复代码,保留一份
- 使用DRY原则(Don't Repeat Yourself)
- 提取公共逻辑为独立函数

**预防措施**:
- ✅ 提交前自查是否有重复代码
- ✅ 使用linter工具检测重复代码
- ✅ Code Review时重点检查

## 🧪 测试规范

### ❌ 错误5: API参数不匹配

**问题描述**:  
测试用例中使用`rating`字段,但实际API使用`feedback`字段,导致测试失败。

**正确做法**:
```python
# 先查看API定义,确认参数名称
@ai_bp.route('/api/ai/feedback', methods=['POST'])
def submit_feedback():
    data = request.get_json()
    # feedback字段,不是rating
    feedback = data['feedback']
```

**预防措施**:
- ✅ 写测试前先查看API定义
- ✅ 确保测试参数与实际API一致
- ✅ 运行测试验证后再提交

## 📚 文档维护

### ❌ 错误6: 文档与代码不同步

**问题描述**:  
代码已修改,但CHANGELOG未更新,或版本号不一致。

**正确做法**:
```
代码修改 → 更新CHANGELOG → Git提交
   ↓           ↓              ↓
 完成       版本号一致      版本号一致
```

**预防措施**:
- ✅ 每次代码变更必须更新CHANGELOG
- ✅ CHANGELOG版本号必须与Git commit一致
- ✅ 提交前检查文档是否完整

## 🔄 工作流程

### ❌ 错误7: 不按标准流程执行

**问题描述**:  
跳过需求确认、方案设计等步骤,直接开始编码。

**标准流程**:
```
阶段1: 产品与功能分析 📋
  ↓
阶段2: 确定开发方向和方案 💡 (提供2-3个方案,等待确认)
  ↓
阶段3: 规划开发任务 📝 (创建任务清单)
  ↓
阶段4: 开发实施(TDD) 🔴🟢🔵
  ↓
阶段5: 执行测试任务 🧪 (覆盖率≥80%)
  ↓
阶段6: 整体验收 ✅ (功能、性能、代码质量、文档)
```

**预防措施**:
- ✅ 严格按照6个阶段执行
- ✅ 每个阶段完成后汇报进度
- ✅ 方案必须等待用户确认
