## 二、Railway 部署（强烈推荐）

### 优势

- ✅ 每月 $5 免费额度（足够小项目使用）
- ✅ 支持 Python Flask
- ✅ 内置 PostgreSQL 数据库
- ✅ GitHub 集成，自动部署
- ✅ 提供自定义域名

### 步骤 1：准备 GitHub 仓库

1. 在 GitHub 创建新仓库（公开或私有均可）
2. 将项目代码推送到仓库：

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

### 步骤 2：注册 Railway

1. 访问 https://railway.app
2. 使用 GitHub 账号登录
3. 首次注册可获得 $5 赠金

### 步骤 3：创建新项目

1. 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择你刚才推送的仓库
4. Railway 会自动检测 Python 项目

### 步骤 4：配置环境变量

在 Railway 项目页面，点击 "Variables" 标签，添加：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| FLASK_ENV | production | 生产环境 |
| SECRET_KEY | 随机密钥 | 见下方生成方法 |
| DATABASE_URL | 自动生成 | Railway 自动提供 |

**生成 SECRET_KEY 的方法**:

```python
import secrets
print(secrets.token_hex(32))
```

### 步骤 5：添加 PostgreSQL 数据库

1. 点击 "New" → "Database" → "Add PostgreSQL"
2. Railway 会自动创建数据库并设置 DATABASE_URL
3. 等待数据库就绪（约 1-2 分钟）

### 步骤 6：代码适配（已完成✅）

项目已配置好 PostgreSQL 支持，`config.py` 中的 `ProductionConfig` 类会自动从环境变量读取 `DATABASE_URL`。

### 步骤 7：提交并部署

```bash
git add .
git commit -m "Add Railway deployment config"
git push
```

Railway 会自动检测到推送并开始部署！

### 步骤 8：查看部署状态

1. 在 Railway 项目页面点击 "Deployments"
2. 查看构建日志，确认无错误
3. 部署成功后，点击 "Generate Domain" 获取访问地址
4. 访问你的应用！

**默认账户**: admin / admin123（或在环境变量中设置 ADMIN_PASSWORD）

## 三、Render 部署（备选方案）

### 优势

- ✅ 永久免费层（有休眠限制）
- ✅ 支持 Python
- ✅ 内置 PostgreSQL

### 步骤 1：注册 Render

访问 https://render.com，使用 GitHub 登录

### 步骤 2：创建 Web Service

1. 点击 "New +" → "Web Service"
2. 连接 GitHub 仓库
3. 配置：
   - **Name**: dog-analytics-demo
   - **Environment**: Python
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python init_production.py && gunicorn app:create_app() --bind 0.0.0.0:$PORT`

### 步骤 3：添加 PostgreSQL

1. New + → Database → PostgreSQL
2. 复制 Internal Database URL
3. 在 Web Service 的环境变量中添加：
   - `DATABASE_URL`=粘贴刚才复制的URL
   - `FLASK_ENV=production`
   - `SECRET_KEY=随机密钥`

### 步骤 4：部署

点击 "Create Web Service"，等待部署完成。

> ⚠️ **注意**: 免费层会在 15 分钟无访问后休眠，首次访问需要等待 30-50 秒唤醒。

## 五、常见问题解答

### Q1: Railway 免费额度够用吗？

**A**: 每月 $5 额度，对于小型 Flask 应用绰绰有余。通常只需 $2-3/月。

### Q2: 数据库会丢失吗？

**A**: Railway 和 Render 的 PostgreSQL 都是持久化的，不会丢失。但建议定期备份。

### Q3: 国内访问速度慢怎么办？

**A**: 可以考虑：
- 使用 Cloudflare CDN（免费）
- 购买国内云服务器（阿里云/腾讯云）
- 接受现状（演示用途可接受）

### Q4: 如何绑定自定义域名？

**A**: Railway 和 Render 都支持自定义域名：
1. 在平台设置中添加你的域名
2. 在域名服务商处配置 CNAME 记录
3. 等待 DNS 生效（最多 48 小时）

### Q5: 如何查看部署日志？

**A**: 
- **Railway**: 项目页面 → Deployments → 点击查看日志
- **Render**: Dashboard → 服务 → Logs 标签

### Q6: 部署后出现 500 错误？

**A**: 检查：
1. 环境变量是否正确设置
2. 数据库连接是否正常
3. 查看部署日志中的错误信息
4. 确保 SECRET_KEY 已设置

### Q7: 如何更新代码？

**A**: 直接推送到 GitHub，Railway/Render 会自动重新部署。

### Q8: 可以多人协作吗？

**A**: 可以！Railway 支持团队协作，邀请成员即可。

### Q9: 数据量大了怎么办？

**A**: 
- Railway: 升级套餐（$5/月起）
- Render: 升级数据库实例
- 或迁移到国内云服务器

### Q10: 如何备份数据库？

**A**: 

**Railway**:
1. 安装 Railway CLI: `npm i -g @railway/cli`
2. 运行: `railway pg backup`

**Render**:
1. Dashboard → Database → Backups
2. 点击 "Create Backup"
