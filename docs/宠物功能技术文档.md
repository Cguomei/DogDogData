# 网页小宠物功能 - 技术文档

## 📋 概述

网页小宠物是一个陪伴用户在浏览网页时的虚拟宠物系统，提供互动、喂食、玩耍等功能，增加用户体验和趣味性。特别适用于页面加载时间较长的场景（如世界地图等大数据可视化页面），为用户提供情感化陪伴。

### 🎉 v3.0.0 重大更新 - 宠物小精灵可爱版
- ✨ **全新形象**: 从 emoji 升级为 CSS 绘制的粉色宠物小精灵
- ✨ **可爱设计**: 类似星之卡比的可爱风格，圆形身体 + 大眼睛 + 腮红
- ✨ **丰富表情**: 根据心情改变眼睛形状（开心/难过/困倦）
- ✨ **流畅动画**: 呼吸、眨眼、弹跳等多种 CSS 动画
- ✨ **性能优化**: 移除复杂 2.5D 旋转，改用 2D 可爱风格
- ✨ **单例增强**: 修复重复显示问题

## 🎯 功能特性

### 1. 核心功能
- ✅ **可爱外观**: 粉色渐变宠物小精灵，CSS 绘制
- ✅ **可触摸互动**: 点击宠物会有弹跳和涟漪特效
- ✅ **可喂食**: 通过喂食按钮降低饥饿度
- ✅ **可玩耍**: 提供抚摸、玩耍等互动选项
- ✅ **动态状态系统**: 包含心情、饥饿度、活力值、亲密度
- ✅ **表情系统**: 眼睛形状随心情变化
- ✅ **睡眠系统**: 宠物可以进入/退出睡眠状态
- ✅ **自动隐藏**: 长时间无操作自动隐藏，显示召唤按钮
- ✅ **状态持久化**: 使用 localStorage 保存宠物状态

### 2. 状态系统

#### 心情状态 (mood)
- `happy` 😊: 开心状态，亲密度高时触发
- `normal` 😐: 普通状态，默认状态
- `sleepy` 😴: 困倦状态，能量低或睡眠时
- `sad` 😢: 不开心状态，饥饿度高时

#### 数值状态
| 状态 | 范围 | 说明 | 变化规律 |
|------|------|------|----------|
| 饥饿度 (hunger) | 0-100 | 0 为饱，100 为饿 | 每 10 秒 +1，离线时每小时 +10 |
| 活力值 (energy) | 0-100 | 活力越高越活跃 | 每 10 秒 -0.5，离线时每小时 -5 |
| 亲密度 (affection) | 0-100 | 互动越多越亲密 | 互动时 +2~5，最高 100 |

## 🔧 使用方法

### 1. 基础集成

宠物系统已自动集成到 `base.html` 模板中，所有继承自 `base.html` 的页面都会自动显示宠物。

```html
<!-- templates/base.html 中已包含 -->
<link rel="stylesheet" href="{{ url_for('static', filename='css/pet.css') }}">
<script src="{{ url_for('static', filename='js/pet.js') }}"></script>
```

### 2. 自定义配置

在页面中创建自定义宠物实例：

```javascript
// 在页面的 extra_scripts 块中
document.addEventListener('DOMContentLoaded', () => {
    window.virtualPet = new VirtualPet({
        containerId: 'virtual-pet-container',
        petName: '团团',      // 宠物名称（默认）
        petType: 'kawaii',    // 宠物类型：kawaii（可爱版）
        autoHideTimeout: 60000  // 自动隐藏时间（毫秒）
    });
});
```

### 3. 手动控制 API

```javascript
// 访问全局实例
const pet = window.virtualPet;

// 控制方法
pet.feed();              // 喂食
pet.play();              // 玩耍
pet.pet();               // 抚摸
pet.toggleSleep();       // 切换睡眠
pet.show();              // 显示宠物
pet.hide();              // 隐藏宠物
pet.showBubble('你好');  // 显示对话气泡

// 状态访问
console.log(pet.state.hunger);    // 饥饿度
console.log(pet.state.mood);      // 心情
console.log(pet.state.energy);    // 活力值
console.log(pet.state.affection); // 亲密度
```

## 🎮 互动动作说明

### 1. 喂食 (🍖)
- **效果**: 饥饿度 -20，活力值 +5
- **限制**: 饥饿度 ≤ 10 时无法喂食
- **动画**: 吃饭摇头动画
- **对话**: 随机显示"好吃的！"、"美味！"等

### 2. 玩耍 (🎾)
- **效果**: 
  - 活力值 -15
  - 饥饿度 +10
  - 心情变为 happy
  - 亲密度 +5
- **限制**: 活力值 < 20 时无法玩耍
- **动画**: 开心摇摆动画
- **对话**: 随机显示"好开心！"、"真好玩！"等

### 3. 抚摸 (❤️)
- **效果**: 
  - 心情变为 happy
  - 亲密度 +3
- **动画**: 开心动画
- **对话**: 随机显示"好舒服~"、"喜欢主人！"等

### 4. 睡觉 (💤)
- **效果**: 切换睡眠状态
- **睡眠中**: 
  - 身体变为蓝色渐变
  - 眼睛眯成一条线
  - 不消耗活力值
  - 不增加饥饿度
- **醒来**: 活力值 +30

### 5. 双击旋转
- **效果**: 360 度旋转动画
- **亲密度**: +10
- **对话**: "哇！转圈圈！"

### 离线计算
当用户再次访问时，系统会根据离线时间自动计算：
- 饥饿度增加：每小时 +10（最多 +50）
- 活力值减少：每小时 -5（最多 -50）

### v3.0.0 故障排除

### 问题 1: 宠物不显示
**检查清单**:
- [ ] 确认 `pet.css?v=3.0.0` 和 `pet.js?v=3.0.0` 文件正确加载
- [ ] 检查浏览器控制台是否有 JavaScript 错误
- [ ] 确认页面继承了 `base.html`
- [ ] 检查是否被其他元素遮挡（z-index）
- [ ] 清除浏览器缓存（Ctrl+F5 强制刷新）

### 问题 2: 状态不保存
**检查清单**:
- [ ] 确认浏览器支持 localStorage
- [ ] 检查浏览器隐私设置是否阻止本地存储
- [ ] 查看控制台是否有存储相关错误
- [ ] 尝试清除 localStorage 重新初始化

### 问题 3: 显示两只宠物
**解决方案**:
- 检查是否有多个 pet.js 加载
- 确认单例模式正常工作
- 查看控制台日志：`window.virtualPetInitialized`

### 问题 4: 动画卡顿
**解决方案**:
```javascript
// 检查是否有其他动画冲突
// 禁用部分动画（不推荐）
.virtual-pet {
    animation: none;
}

// 或降低动画质量
.pet-body-kawaii {
    animation-duration: 5s; // 减慢呼吸速度
}
```

### 问题 5: 表情不变化
**检查**:
- 打开控制台查看心情日志
- 检查 `updateMoodDisplay()` 方法
- 确认心情状态值正确

##  性能优化

### v3.0.0 优化措施
1. **移除 2.5D 旋转**: 删除复杂的旋转逻辑和 mousemove 监听
2. **CSS 绘制**: 不再依赖 emoji 或图片，零 HTTP 请求
3. **单例模式**: 防止重复初始化和按钮重复创建
4. **延迟加载**: 页面加载后 500ms 初始化
5. **GPU 加速**: 使用 CSS transform 和 animation
6. **事件优化**: 减少不必要的事件监听

### 代码体积
- CSS: ~640 行（包含详细注释）
- JS: ~720 行（包含详细注释）
- 总体积比 v2.5.1 减少约 30%

### 浏览器性能
- 60 FPS 流畅动画
- 低内存占用
- 支持所有现代浏览器
