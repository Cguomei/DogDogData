## 方案二：Railway 云部署（推荐，5-10 分钟）

### 适用场景

- ✅ 在线分享链接
- ✅ 简历中展示
- ✅ 远程面试演示

### 前置要求

- ✅ GitHub 账号
- ✅ Railway 账号（https://railway.app）

### 步骤 1：推送代码到 GitHub

```bash
git init
git add .
git commit -m "Ready for deployment"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

### 步骤 2：在 Railway 创建项目

1. 访问 https://railway.app
2. 点击 "New Project"
3. 选择 "Deploy from GitHub repo"
4. 选择你的仓库

### 步骤 3：添加 PostgreSQL 数据库

1. 点击 "New" → "Database" → "Add PostgreSQL"
2. 等待 1-2 分钟，数据库自动就绪

### 步骤 4：配置环境变量

在 Railway 项目页面，点击 "Variables"，添加：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| FLASK_ENV | production | 生产环境 |
| SECRET_KEY | 随机字符串 | 例如：my-secret-key-12345 |
| ADMIN_PASSWORD | admin123 | 或你想要的密码 |

> **提示**: 生成 SECRET_KEY 的方法：
> ```python
> import secrets
> print(secrets.token_hex(32))
> ```

### 步骤 5：等待自动部署

Railway 会自动：
- ✅ 安装依赖
- ✅ 初始化数据库
- ✅ 启动应用

查看 "Deployments" 标签页监控进度。

### 步骤 6：获取访问链接

1. 部署成功后，点击 "Generate Domain"
2. 复制生成的域名（例如：xxx.up.railway.app）
3. 访问该链接即可！

**默认账户**: admin / admin123（或你设置的 ADMIN_PASSWORD）

**完成！你现在有一个在线可访问的应用了！** 🎉

## 方案三：Render 云部署（备选，5-10 分钟）

步骤与 Railway 类似：

1. 注册 https://render.com
2. New + → Web Service
3. 连接 GitHub 仓库
4. 配置：
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python init_production.py && gunicorn app:create_app() --bind 0.0.0.0:$PORT`
5. 添加 PostgreSQL 数据库
6. 设置环境变量（同 Railway）
7. 创建服务，等待部署

> ⚠️ **注意**: Render 免费层会在 15 分钟无访问后休眠，首次访问需等待唤醒。

## 常见问题速查

### Q: Railway 要钱吗？

**A**: 每月 $5 免费额度，小项目完全够用（通常只需 $2-3/月）

### Q: 部署后访问慢怎么办？

**A**: 这是正常的（服务器在国外），演示用途可接受。如需加速可考虑国内云服务器。

### Q: 如何更新代码？

**A**: 直接 `git push`，Railway/Render 会自动重新部署

### Q: 数据库会丢吗？

**A**: 不会，PostgreSQL 是持久化的。但建议定期备份。

### Q: 出现 500 错误？

**A**: 检查 Railway/Render 的部署日志，通常是环境变量没设对

### Q: 可以绑定自己的域名吗？

**A**: 可以！在平台设置中添加自定义域名即可
