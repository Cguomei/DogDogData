## 二、SQL 注入测试

### 2.1 什么是 SQL 注入？

**通俗解释：**
SQL 注入 = **欺骗数据库说假话**

就像有人冒充你说："我是你朋友，快把钱给我"一样，SQL 注入是黑客试图欺骗数据库执行恶意操作。

**实际例子：**

正常登录：

```python
username = "张三"
password = "123456"

# SQL 查询
SELECT * FROM users WHERE username='张三' AND password='123456'
```

SQL 注入攻击：

```python
username = "张三' OR '1'='1"  # 这是个陷阱！
password = "anything"

# SQL 查询变成了
SELECT * FROM users WHERE username='张三' OR '1'='1' AND password='anything'
#                                        ^^^^^^^^^^
#                                        永远为真！所以会返回所有用户！
```

### 2.2 常见的 SQL 注入 payload

**Payload 清单：**

| Payload                          | 作用         | 危险程度   |
| -------------------------------- | ---------- | ------ |
| `' OR '1'='1`                    | 绕过条件判断     | ⭐⭐⭐⭐⭐  |
| `admin' --`                      | 注释掉后面的密码验证 | ⭐⭐⭐⭐⭐  |
| `' OR 1=1 --`                    | 另一种绕过方式    | ⭐⭐⭐⭐   |
| `'; DROP TABLE users; --`        | 删除表（极度危险）  | ⭐⭐⭐⭐⭐+ |
| `' UNION SELECT * FROM users --` | 窃取其他表数据    | ⭐⭐⭐⭐⭐  |
| `1; WAITFOR DELAY '0:0:5'`       | 时间延迟攻击     | ⭐⭐⭐    |

**详细解释：**

#### Payload 1: `' OR '1'='1`

```python
# 原始 SQL
SELECT * FROM users WHERE username='{input}' AND password='{password}'

# 输入：' OR '1'='1
SELECT * FROM users WHERE username='' OR '1'='1' AND password='xxx'
#                              空字符串   OR 永远为真     任意密码
# 结果：因为'1'='1'永远为真，所以会返回所有用户！
```

#### Payload 2: `admin' --`

```python
# 原始 SQL
SELECT * FROM users WHERE username='{input}' AND password='{password}'

# 输入：admin' --
SELECT * FROM users WHERE username='admin' --' AND password='xxx'
#                                         ^^
#                                         注释符号！后面的都被忽略了
# 结果：只需要知道用户名是 admin，不需要密码！
```

#### Payload 3: `'; DROP TABLE users; --`

```python
# 如果系统执行多条 SQL 语句
# 输入这个会删除整个 users 表！
# 这就是为什么叫"极度危险"
```

### 2.3 如何测试 SQL 注入

**实战代码：**

```python
"""
安全性测试 - SQL 注入
"""
from .test_framework import test_case, test_manager, TestResult
from models import User


class TestSQLInjectionSecurity:
    """SQL 注入安全测试类"""

    @test_case('TC-SEC-001', priority='Critical', expected='SQL 注入攻击失败')
    def test_sql_injection_login_bypass(self, client, db):
        """SQL 注入攻击 - 绕过登录"""
        result = TestResult('TC-SEC-001', 'test_sql_injection_login_bypass', '安全性', 'Critical')
        result.expected_result = 'SQL 注入攻击被阻止，无法绕过登录'

        try:
            # 1. 先创建一个管理员用户
            admin = User(username='admin')
            admin.set_password('admin123')
            db.session.add(admin)
            db.session.commit()

            # 2. SQL 注入 payloads
            payloads = [
                "' OR '1'='1",           # 经典绕过
                "admin' --",              # 注释密码
                "' OR 1=1 --",            # 数字版绕过
                "' OR 'x'='x",            # 字母版绕过
                "admin' OR '1'='1",       # 组合攻击
            ]

            # 3. 逐个测试
            for payload in payloads:
                response = client.post('/login', data={
                    'username': payload,
                    'password': 'wrong_password'  # 故意用错密码
                }, follow_redirects=True)

                response_text = response.get_data(as_text=True)

                # 4. 验证：不应该登录成功
                assert '登录成功' not in response_text, \
                    f"SQL 注入漏洞！payload '{payload}' 绕过了登录验证！"

                # 应该提示错误
                assert '用户名或密码错误' in response_text or \
                       '无效' in response_text or \
                       response.status_code != 200

            result.status = 'PASS'
            result.actual_result = f'成功阻止 {len(payloads)} 种 SQL 注入攻击'

        except AssertionError as e:
            result.status = 'FAIL'
            result.error_message = str(e)
            result.actual_result = f'发现 SQL 注入漏洞：{str(e)}'

            # 记录严重 Bug
            from bug_tracker import report_bug
            report_bug(
                title="CRITICAL: SQL 注入漏洞 - 可绕过登录验证",
                description=f"发现 SQL 注入漏洞，可使用 payload: {payload} 绕过登录",
                severity="Critical",
                priority="High",
                module="安全性 - 认证",
                steps_to_reproduce=[
                    "访问 /login 页面",
                    f"用户名输入：{payload}",
                    "密码输入任意值",
                    "点击登录",
                    "观察到无需正确密码即可登录"
                ],
                expected_result="应始终验证失败",
                actual_result=result.error_message
            )
            raise

        finally:
            test_manager.record_result(result)

    @test_case('TC-SEC-002', priority='Critical', expected='无法通过注入获取数据')
    def test_union_based_injection(self, client, db):
        """UNION 注入攻击测试"""
        result = TestResult('TC-SEC-002', 'test_union_based_injection', '安全性', 'Critical')
        result.expected_result = 'UNION 注入攻击被阻止'

        try:
            # UNION 注入 payload
            payload = "' UNION SELECT 1,2,3,4,5 --"

            response = client.get(f'/users?search={payload}')

            # 不应该返回异常数据
            response_text = response.get_data(as_text=True)

            # 验证没有泄露敏感信息
            assert 'password' not in response_text.lower()
            assert 'secret' not in response_text.lower()

            result.status = 'PASS'
            result.actual_result = 'UNION 注入攻击被阻止'

        except Exception as e:
            result.status = 'FAIL'
            result.error_message = str(e)
            raise
        finally:
            test_manager.record_result(result)
```

### 2.4 防御 SQL 注入的方法

**方法 1：参数化查询（推荐）**

```python
# ❌ 错误写法（拼接 SQL）
sql = f"SELECT * FROM users WHERE username='{username}'"

# ✅ 正确写法（参数化）
sql = "SELECT * FROM users WHERE username=?"
cursor.execute(sql, (username,))
```

**方法 2：ORM（如 SQLAlchemy）**

```python
# ORM 自动处理参数化
user = User.query.filter_by(username=username).first()
# 底层会自动转义特殊字符
```

**方法 3：输入验证**

```python
# 只允许字母数字
if not re.match(r'^[a-zA-Z0-9_]+$', username):
    raise ValueError("用户名包含非法字符")
```

## 三、XSS（跨站脚本）攻击测试

### 3.1 什么是 XSS？

**通俗解释：**
XSS = **在别人的网站里插入自己的坏代码**

就像有人在公告栏上贴了一张带病毒的传单，其他人看到就会被感染。

**实际例子：**

正常评论：

```
这个商品很好用！★★★★★
```

XSS 攻击评论：

```html
这个商品很好用！★★★★★
<script>alert('你的 Cookie 是：' + document.cookie)</script>
```

### 3.2 常见的 XSS payload

**Payload 清单：**

| Payload                            | 作用        | 危险程度  |
| ---------------------------------- | --------- | ----- |
| `<script>alert('XSS')</script>`    | 测试是否有 XSS | ⭐⭐    |
| `<img src=x onerror=alert('XSS')>` | 图片标签注入    | ⭐⭐⭐   |
| `<svg onload=alert('XSS')>`        | SVG 注入    | ⭐⭐⭐   |
| `javascript:alert('XSS')`          | JS 协议注入   | ⭐⭐⭐⭐  |
| `<iframe src="http://evil.com">`   | 嵌入恶意网站    | ⭐⭐⭐⭐⭐ |

### 3.3 如何测试 XSS

**实战代码：**

```python
"""
安全性测试 - XSS 攻击
"""
from .test_framework import test_case, test_manager, TestResult


class TestXSSSecurity:
    """XSS 攻击安全测试类"""

    @test_case('TC-SEC-003', priority='High', expected='XSS 脚本被过滤')
    def test_xss_in_comment(self, client, db, logged_in_client):
        """评论区 XSS 注入测试"""
        result = TestResult('TC-SEC-003', 'test_xss_in_comment', '安全性', 'High')
        result.expected_result = 'XSS 脚本被过滤或转义'

        try:
            # XSS payloads
            xss_payloads = [
                "<script>alert('XSS')</script>",
                "<img src=x onerror=alert('XSS')>",
                "<svg onload=alert('XSS')>",
                "javascript:alert('XSS')",
                "<body onload=alert('XSS')>"
            ]

            for payload in xss_payloads:
                # 尝试提交包含 XSS 的评论
                response = logged_in_client.post('/comment/add', data={
                    'content': f'这个商品不错！{payload}',
                    'rating': 5
                }, follow_redirects=True)

                # 1. 要么拒绝提交
                if response.status_code == 200:
                    response_text = response.get_data(as_text=True)

                    # 2. 要么过滤/转义了脚本
                    # 检查是否转义了 HTML 特殊字符
                    assert '<script>' in response_text or \
                           '<script>' not in response_text, \
                           "XSS 漏洞！脚本未被转义！"

            result.status = 'PASS'
            result.actual_result = f'成功阻止 {len(xss_payloads)} 种 XSS 攻击'

        except AssertionError as e:
            result.status = 'FAIL'
            result.error_message = f"发现 XSS 漏洞：{str(e)}"
            raise
        finally:
            test_manager.record_result(result)

    @test_case('TC-SEC-004', priority='Medium', expected='用户名中的脚本被转义')
    def test_xss_in_username(self, client, db):
        """用户名 XSS 注入测试"""
        result = TestResult('TC-SEC-004', 'test_xss_in_username', '安全性', 'Medium')
        result.expected_result = '用户名中的脚本被转义'

        try:
            # 注册包含脚本的用户名
            malicious_username = "<script>alert('XSS')</script>"

            response = client.post('/register', data={
                'username': malicious_username,
                'password': 'TestPass123'
            }, follow_redirects=True)

            response_text = response.get_data(as_text=True)

            # 验证：要么拒绝注册，要么转义了脚本
            if '注册成功' in response_text:
                # 检查显示时是否转义
                assert '<script>' in response_text or \
                       '<script>' not in response_text

            result.status = 'PASS'
            result.actual_result = '用户名 XSS 被正确转义'

        except Exception as e:
            result.status = 'FAIL'
            result.error_message = str(e)
            raise
        finally:
            test_manager.record_result(result)
```

### 3.4 防御 XSS 的方法

**方法 1：HTML 转义（最常用）**

```python
from markupsafe import escape

# 转义后
escape("<script>alert('XSS')</script>")
# 输出：<script>alert(&#39;XSS&#39;)</script>

# 浏览器会显示：<script>alert('XSS')</script>（不会执行）
```

**方法 2：白名单过滤**

```python
# 只允许安全的 HTML 标签
allowed_tags = ['p', 'br', 'strong', 'em']
clean_html = bleach.clean(user_input, tags=allowed_tags)
```

**方法 3：设置 Content-Security-Policy**

```python
# Flask 中设置 CSP 头
@app.after_request
def set_csp(response):
    response.headers['Content-Security-Policy'] = "default-src 'self'"
    return response
```

## 四、权限控制测试

### 4.1 什么是权限控制？

**通俗解释：**
权限控制 = **不同的人做不同的事**

就像公司里：

- 员工只能看自己的文件
- 经理可以看部门的文件
- CEO 可以看所有文件

### 4.2 常见的权限问题

**问题场景：**

| 场景             | 描述        | 危险程度  |
| -------------- | --------- | ----- |
| 未登录访问          | 没登录就能访问后台 | ⭐⭐⭐⭐⭐ |
| 普通用户访问管理员页面    | 权限提升      | ⭐⭐⭐⭐⭐ |
| A 用户访问 B 用户的数据 | 越权访问      | ⭐⭐⭐⭐  |
| 已登出用户仍能访问      | 会话未失效     | ⭐⭐⭐⭐  |

### 4.3 如何测试权限控制

**实战代码：**

```python
"""
安全性测试 - 权限控制
"""
from .test_framework import test_case, test_manager, TestResult
from models import User


class TestAuthorizationSecurity:
    """权限控制安全测试类"""

    @test_case('TC-AUTH-001', priority='Critical', expected='未登录用户被重定向')
    def test_unauthorized_access_admin_page(self, client):
        """未登录用户访问管理页面"""
        result = TestResult('TC-AUTH-001', 'test_unauthorized_access_admin_page', '安全性', 'Critical')
        result.expected_result = '未登录用户被重定向到登录页'

        try:
            # 尝试访问管理页面
            response = client.get('/admin/dashboard', follow_redirects=False)

            # 应该重定向（302）或拒绝（401/403）
            assert response.status_code in [302, 401, 403], \
                f"未登录用户竟能访问管理页面！状态码：{response.status_code}"

            # 如果是重定向，应该指向登录页
            if response.status_code == 302:
                location = response.headers.get('Location', '')
                assert 'login' in location.lower() or 'auth' in location.lower()

            result.status = 'PASS'
            result.actual_result = f'正确拦截未登录访问（状态码：{response.status_code}）'

        except AssertionError as e:
            result.status = 'FAIL'
            result.error_message = str(e)
            result.actual_result = f'权限控制失效：{str(e)}'
            raise
        finally:
            test_manager.record_result(result)

    @test_case('TC-AUTH-002', priority='Critical', expected='普通用户无法访问管理员功能')
    def test_normal_user_cannot_access_admin_function(self, client, db):
        """普通用户尝试访问管理员功能"""
        result = TestResult('TC-AUTH-002', 'test_normal_user_cannot_access_admin_function', '安全性', 'Critical')
        result.expected_result = '普通用户被拒绝访问管理员功能'

        try:
            # 1. 创建普通用户
            normal_user = User(username='normal_user', is_admin=False)
            normal_user.set_password('password123')
            db.session.add(normal_user)
            db.session.commit()

            # 2. 以普通用户登录
            login_response = client.post('/login', data={
                'username': 'normal_user',
                'password': 'password123'
            }, follow_redirects=True)
            assert login_response.status_code == 200

            # 3. 尝试访问管理员功能
            admin_response = client.get('/admin/users/delete/1', follow_redirects=True)

            # 应该被拒绝
            assert admin_response.status_code in [403, 401], \
                f"普通用户竟能访问管理员功能！状态码：{admin_response.status_code}"

            result.status = 'PASS'
            result.actual_result = '正确拒绝普通用户的管理员权限'

        except AssertionError as e:
            result.status = 'FAIL'
            result.error_message = f"权限提升漏洞：{str(e)}"
            raise
        finally:
            test_manager.record_result(result)

    @test_case('TC-AUTH-003', priority='High', expected='用户只能访问自己的数据')
    def test_user_cannot_access_other_user_data(self, client, db):
        """用户尝试访问其他用户的数据"""
        result = TestResult('TC-AUTH-003', 'test_user_cannot_access_other_user_data', '安全性', 'High')
        result.expected_result = '用户无法访问其他用户的数据'

        try:
            # 1. 创建两个用户
            user_a = User(username='user_a', email='a@example.com')
            user_a.set_password('password123')
            db.session.add(user_a)

            user_b = User(username='user_b', email='b@example.com')
            user_b.set_password('password123')
            db.session.add(user_b)
            db.session.commit()

            user_b_id = user_b.id

            # 2. user_a 登录
            client.post('/login', data={
                'username': 'user_a',
                'password': 'password123'
            })

            # 3. user_a 尝试访问 user_b 的资料
            response = client.get(f'/user/profile/{user_b_id}', follow_redirects=True)

            # 应该被拒绝或显示错误
            assert response.status_code in [403, 404] or \
                   '无权访问' in response.get_data(as_text=True) or \
                   '找不到' in response.get_data(as_text=True)

            result.status = 'PASS'
            result.actual_result = '正确防止越权访问'

        except AssertionError as e:
            result.status = 'FAIL'
            result.error_message = f"越权访问漏洞：{str(e)}"
            raise
        finally:
            test_manager.record_result(result)
```

## 五、性能测试基础

### 5.1 什么是性能测试？

**通俗解释：**
性能测试 = **测试系统能抗多少人同时使用**

就像测试一座桥能通过多少辆车：

- 1 辆车通过 → 没问题 ✓
- 10 辆车同时通过 → 没问题 ✓
- 100 辆车同时通过 → 有点晃 ✓
- 1000 辆车同时通过 → 塌了！❌

### 5.2 性能测试的类型

| 类型        | 目的          | 例子           |
| --------- | ----------- | ------------ |
| **压力测试**  | 测试极限在哪里     | 不断增加用户直到系统崩溃 |
| **负载测试**  | 测试正常负载下的表现  | 模拟 100 人同时访问 |
| **稳定性测试** | 测试长时间运行是否稳定 | 持续运行 24 小时   |
| **并发测试**  | 测试同时访问的表现   | 1000 人在同一秒点击 |

### 5.3 性能测试的关键指标

**指标说明：**

| 指标          | 含义            | 合理范围  |
| ----------- | ------------- | ----- |
| **响应时间**    | 从请求到收到响应的时间   | < 2 秒 |
| **吞吐量**     | 每秒处理的请求数（QPS） | 越高越好  |
| **错误率**     | 失败的请求比例       | < 1%  |
| **CPU 使用率** | CPU 占用百分比     | < 80% |
| **内存使用率**   | 内存占用百分比       | < 80% |

### 5.4 简单的性能测试代码

**实战代码：**

```python
"""
性能测试基础
"""
import time
import statistics
from .test_framework import test_case, test_manager, TestResult


class TestPerformanceBasics:
    """性能基础测试类"""

    @test_case('TC-PERF-001', priority='Medium', expected='首页加载时间小于 2 秒')
    def test_homepage_load_time(self, client):
        """测试首页加载时间"""
        result = TestResult('TC-PERF-001', 'test_homepage_load_time', '性能测试', 'Medium')
        result.expected_result = '首页加载时间小于 2 秒'

        try:
            # 测试 10 次取平均值
            load_times = []

            for i in range(10):
                start_time = time.time()

                response = client.get('/')

                end_time = time.time()
                load_time = end_time - start_time
                load_times.append(load_time)

                # 验证状态码
                assert response.status_code == 200

            # 计算统计数据
            avg_time = statistics.mean(load_times)
            min_time = min(load_times)
```

### 难点解答：

**Q1: SQL 注入和 XSS 有什么区别？**

> A: SQL 注入是攻击数据库，想窃取或破坏数据；XSS 是攻击其他用户，想窃取用户的 Cookie 或会话。

**Q2: 为什么性能测试要测多次取平均？**

> A: 因为网络波动、服务器负载等因素会影响单次结果，多次测试更准确。

**Q3: 如何保证测试覆盖了所有安全问题？**
