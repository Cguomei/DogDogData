## 一、核心业务流程

### 1.1 用户注册流程

#### 流程图

```mermaid
graph TB
    A[访问注册页面] --> B[填写用户名和密码]
    B --> C{表单验证}
    C -->|格式错误 | D[显示错误提示]
    D --> B
    C -->|格式正确 | E{检查用户名唯一性}
    E -->|已存在 | F[提示用户名被占用]
    F --> B
    E -->|可用 | G[密码加密]
    G --> H[保存到数据库]
    H --> I[注册成功]
    I --> J[跳转到登录页]
```

#### 业务规则

1. **用户名规则**
   - 长度：3-20 个字符
   - 字符集：中文、字母、数字、下划线
   - 唯一性：全局唯一

2. **密码规则**
   - 最小长度：6 位
   - 字符类型：不限（字母/数字/特殊字符均可）
   - 加密方式：Werkzeug password_hash

3. **异常处理**
   - 网络错误：友好提示"请稍后重试"
   - 数据库错误：记录日志，提示用户"系统繁忙"

### 1.2 用户登录流程

#### 流程图

```mermaid
graph TB
    A[访问登录页面] --> B[输入用户名和密码]
    B --> C[提交表单]
    C --> D{CSRF 验证}
    D -->|失败 | E[拒绝请求]
    D -->|通过 | F[查询用户]
    F --> G{用户存在？}
    G -->|否 | H[提示用户名或密码错误]
    G -->|是 | I{密码正确？}
    I -->|否 | H
    I -->|是 | J[创建会话]
    J --> K{有 next 参数？}
    K -->|是 | L[跳转到 next 页面]
    K -->|否 | M[跳转到首页]
    H --> N[返回登录页]
    N --> B
```

#### 业务规则

1. **安全策略**
   - 不明确指出用户名或密码错误（防枚举攻击）
   - 使用 ORM 参数化查询（防 SQL 注入）
   - Session 管理：HttpOnly Cookie

2. **会话管理**
   - 登录成功后创建 session
   - session 中保存 user_id 和 role
   - 登出时销毁 session

3. **跳转逻辑**
   - 如果有 next 参数，登录后跳转到该页面
   - 否则跳转到首页
   - 禁止跳转到外部链接（防钓鱼）

#### 1.3.2 批量导入流程

```mermaid
graph TB
    A[选择文件] --> B{文件格式检查}
    B -->|不支持 | C[提示格式错误]
    B -->|支持 | D[解析文件]
    D --> E{解析成功？}
    E -->|失败 | F[提示文件解析失败]
    E -->|成功 | G{检查列完整性}
    G -->|缺少列 | H[提示缺少必要列]
    G -->|完整 | I[逐行处理数据]
    I --> J{数据验证}
    J -->|失败 | K[记录失败原因]
    J -->|通过 | L[保存到数据库]
    L --> M{是否唯一冲突？}
    M -->|是 | N[跳过并记录]
    M -->|否 | O[成功插入]
    N --> P[继续下一行]
    O --> P
    P --> Q{处理完成？}
    Q -->|否 | I
    Q -->|是 | R[统计结果]
```

#### 业务规则

1. **数据验证**
   - 品种名：必填，2-100 字符，禁止 HTML 标签
   - 平均寿命：可选，0-100 岁，数值类型
   - 体型类别：可选，枚举值验证
   - 人气值：可选，0-1000 整数

2. **错误处理**
   - 单条失败不影响其他数据
   - 详细记录每条失败原因
   - 返回友好的错误提示

## 三、数据处理流程

### 3.1 数据来源

```
数据源
├─ jd_dogs 表 (狗狗基本信息)
│   ├─ dog_name (品种名)
│   ├─ price (价格)
│   ├─ weight (体重)
│   ├─ pet_level (级别)
│   └─ shop_name (店铺)
│
├─ dog_price 表 (价格详情)

    ├─ food_name (品牌名)
    ├─ price (价格)
```

### 3.2 数据统计流程

```mermaid
graph TB
    A[开始统计] --> B[连接数据库]
    B --> C[执行 COUNT 查询]
    C --> D[获取狗狗总数]
    D --> E[执行 AVG 查询]
    E --> F[获取平均价格]
```

### 3.3 数据质量控制

1. **完整性检查**

   - 关联字段一致性

2. **准确性保证**
   - 使用事务保证一致性
   - 异常回滚机制
   - 数据校验日志

3. **时效性管理**
   - 定时任务自动更新
   - 缓存过期策略
   - 手动刷新接口

## 四、错误处理流程

### 4.1 错误分类

| 错误类型 | HTTP 状态码 | 处理方式 |
|---------|-----------|---------|
| 客户端错误 | 400 | 返回具体错误信息 |
| 未授权 | 401 | 重定向登录 |
| 禁止访问 | 403 | 显示无权限 |
| 资源不存在 | 404 | 友好提示页 |
| 服务器错误 | 500 | 记录日志，通用提示 |

### 4.2 错误处理流程

```mermaid
graph TB
    A[请求到达] --> B[执行业务逻辑]
    B --> C{发生异常？}
    C -->|否 | D[正常响应]
    C -->|是 | E{异常类型}
    E -->|验证错误 | F[返回 400+ 错误详情]
    E -->|认证失败 | G[返回 401 或重定向]
    E -->|权限不足 | H[返回 403]
    E -->|资源缺失 | I[返回 404]
    E -->|系统错误 | J[记录日志]
    J --> K[返回 500+ 通用提示]
    F --> L[前端显示错误]
    G --> L
    H --> L
    I --> L
    K --> L
```

### 4.3 错误日志记录

## 五、安全防护流程

### 5.1 CSRF 防护流程

```mermaid
graph TB
    A[用户访问表单页面] --> B[生成 CSRF token]
    B --> C[保存到 session]
    C --> D[渲染到隐藏字段]
    D --> E[用户提交表单]
    E --> F{提取 token}
    F --> G{验证 token}
    G -->|无效 | H[拒绝请求]
    G -->|有效 | I{检查过期？}
    I -->|是 | H
    I -->|否 | J[处理业务]
```

### 5.2 XSS 防护流程

```
用户输入
  ↓
过滤 HTML 标签 (正则表达式)
  ↓
转义特殊字符 (& < > " ')
  ↓
保存到数据库
  ↓
输出时 Jinja2 自动转义
  ↓
浏览器安全显示
```

### 5.3 SQL 注入防护

```mermaid
graph TB
    A[接收用户输入] --> B[参数化查询准备]
    B --> C[SQL 语句带占位符]
    C --> D[传入参数值]
    D --> E[PyMySQL 处理]
    E --> F[执行查询]
    F --> G[返回结果]
    
    style D fill:#90EE90
    style E fill:#90EE90
```

**示例对比**:

❌ **危险做法**:
```python
sql = f"SELECT * FROM users WHERE username='{username}'"
```

✅ **安全做法**:
```python
cur.execute("SELECT * FROM users WHERE username=%s", (username,))
```

## 六、性能优化流程

### 6.1 缓存流程

```mermaid
graph TB
    A[收到请求] --> B{检查缓存}
    B -->|命中 | C[返回缓存数据]
    B -->|未命中 | D[查询数据库]
    D --> E[处理数据]
    E --> F[保存到缓存]
    F --> G[返回响应]
    
    style B fill:#FFD700
    style C fill:#90EE90
```

### 6.2 数据库优化

1. **索引策略**
   - 主键索引：id
   - 唯一索引：breed_name, username
   - 复合索引：常用查询组合

2. **查询优化**
   - 避免 SELECT *
   - 使用 LIMIT 限制结果数
   - 避免 N+1 查询

3. **连接池**
   - 复用数据库连接
   - 设置最大连接数
   - 空闲连接超时回收
