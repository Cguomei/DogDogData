## 二、运行测试

### 2.1 快速开始

```bash
# 激活虚拟环境
fastApiProject\Scripts\activate

# 运行所有测试
pytest Test/ -v

# 运行特定测试文件
pytest Test/test_auth.py -v

# 运行特定测试函数
pytest Test/test_auth.py::test_login_success -v

# 生成 HTML 报告
pytest Test/ --html=reports/test_report.html --self-contained-html

# 生成覆盖率报告
pytest Test/ --cov=. --cov-report=html
```

### 2.2 常用命令

| 命令                    | 说明             |
| --------------------- | -------------- |
| `pytest`              | 运行所有测试         |
| `pytest -v`           | 详细输出           |
| `pytest -x`           | 遇到第一个失败就停止     |
| `pytest -k "keyword"` | 运行包含关键字的测试     |
| `pytest --tb=short`   | 简短的错误追踪        |
| `pytest -m "slow"`    | 运行标记为 slow 的测试 |

## 三、编写测试用例

### 3.1 基本结构

```python
import pytest
from app import create_app
from models import db, User

class TestAuthentication:
    """用户认证测试类"""

    @pytest.fixture(autouse=True)
    def setup(self):
        """每个测试前的设置"""
        self.app = create_app('testing')
        self.client = self.app.test_client()

        with self.app.app_context():
            db.create_all()

        yield

        # 清理
        with self.app.app_context():
            db.drop_all()

    def test_register_success(self):
        """测试注册成功"""
        response = self.client.post('/register', data={
            'username': 'testuser',
            'password': 'test123456'
        }, follow_redirects=True)

        assert response.status_code == 200
        assert b'注册成功' in response.data

    def test_login_with_wrong_password(self):
        """测试错误密码登录"""
        response = self.client.post('/login', data={
            'username': 'admin',
            'password': 'wrongpassword'
        }, follow_redirects=True)

        assert response.status_code == 200
        assert b'用户名或密码错误' in response.data
```

### 3.3 使用 Fixtures

```python
# conftest.py
import pytest
from app import create_app
from models import db, User

@pytest.fixture
def app():
    """创建测试应用"""
    app = create_app('testing')
    return app

@pytest.fixture
def client(app):
    """创建测试客户端"""
    return app.test_client()

@pytest.fixture
def admin_user(client):
    """创建已登录的管理员"""
    client.post('/login', data={
        'username': 'admin',
        'password': 'admin123'
    })
    return client

@pytest.fixture
def sample_breed(app):
    """创建示例品种"""
    from models_extended import DogBreed

    with app.app_context():
        breed = DogBreed(
            breed_name='金毛寻回犬',
            avg_life_years=12
        )
        db.session.add(breed)
        db.session.commit()
        yield breed
        db.session.delete(breed)
        db.session.commit()
```

## 四、UI 自动化测试

### 4.1 Playwright 示例

```python
from playwright.sync_api import sync_playwright

def test_homepage_loads():
    """测试首页加载"""
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=False)
        page = browser.new_page()

        # 访问首页
        page.goto('http://localhost:5000/')

        # 验证标题
        assert '狗狗数据分析系统' in page.title()

        # 验证核心指标卡片存在
        cards = page.query_selector_all('.stat-card')
        assert len(cards) == 8

        browser.close()

def test_chart_navigation():
    """测试图表导航"""
    with sync_playwright() as p:
        browser = p.chromium.launch()
        page = browser.new_page()

        page.goto('http://localhost:5000/')

        # 点击图表链接
        page.click('a[href="/charts"]')

        # 等待页面加载
        page.wait_for_selector('.chart-grid')

        # 验证图表列表显示
        charts = page.query_selector_all('.chart-item')
        assert len(charts) > 0

        browser.close()
```

## 七、测试最佳实践

### 7.1 命名规范

- 测试文件: `test_<module>.py`
- 测试类: `Test<Module>`
- 测试函数: `test_<functionality>_<scenario>`

示例：

```python
class TestAuthentication:
    def test_login_success(self):
    def test_login_wrong_password(self):
    def test_register_duplicate_username(self):
```

### 7.2 AAA 模式

每个测试遵循 **Arrange-Act-Assert** 模式：

```python
def test_example(self):
    # Arrange - 准备
    user_data = {'username': 'test', 'password': 'test123'}

    # Act - 执行
    response = self.client.post('/register', data=user_data)

    # Assert - 断言
    assert response.status_code == 200
```

### 7.3 测试隔离

- 每个测试独立，不依赖其他测试的执行顺序
- 使用 fixtures 管理测试数据
- 测试后清理数据（使用 teardown）

### 7.4 有意义的断言

```python
# ❌ 不好的断言
assert response.status_code == 200

# ✅ 好的断言
assert response.status_code == 200, f"期望 200，实际 {response.status_code}"
assert b'注册成功' in response.data, "未找到成功消息"
```

## 八、持续集成

### 8.1 GitHub Actions 示例

```yaml
# .github/workflows/test.yml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.9'

    - name: Install dependencies
      run: |
        pip install -r requirements.txt
        pip install pytest pytest-cov pytest-html

    - name: Run tests
      run: |
        pytest Test/ -v --cov=. --cov-report=xml --html=reports/test_report.html

    - name: Upload coverage
      uses: codecov/codecov-action@v2
      with:
        file: ./coverage.xml
```

## 九、常见问题

### Q1: 测试数据库连接失败？

**A**: 确保使用测试配置：

```python
app = create_app('testing')
```

### Q2: Session 相关问题？

**A**: 使用 `follow_redirects=True` 自动处理重定向。

### Q3: 外键约束错误？

**A**: 按正确顺序删除数据，或使用 `db.drop_all()` 清理。
