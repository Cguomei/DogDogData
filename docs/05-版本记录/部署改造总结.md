## 二、核心功能说明

### 1. 演示模式（Demo Mode）

#### 特点

- 🔹 使用 SQLite 数据库（文件型，无需安装 MySQL）
- 🔹 自动包含完整的演示数据
- 🔹 一键启动，零配置
- 🔹 适合本地展示和面试演示

#### 使用方法

**方法 1**：双击 `start_demo.bat`

**方法 2**：
```bash
python seed_demo_data.py
set FLASK_ENV=demo
python app.py
```

#### 演示数据包括

- 👥 3 个用户账户（admin/demo/test）
- 🐕 18 种热门犬种
- 📊 用户资料和收藏数据
- 📈 仪表盘统计数据

### 2. 云部署支持

#### 支持平台

- 🚀 **Railway**（推荐）- 每月 $5 免费额度
- 🌐 **Render** - 永久免费层（有休眠）
- ☁️ 其他支持 Python 的平台

#### 关键改进

- ✅ 支持 PostgreSQL 数据库（从环境变量读取 DATABASE_URL）
- ✅ 使用 Gunicorn 作为生产服务器
- ✅ 自动数据库初始化（init_production.py）
- ✅ 完善的部署文档

## 六、常见问题

### Q1: demo.db 文件应该提交到 Git 吗？

**A**: 不应该。`.gitignore` 已配置忽略该文件。每次部署时运行 `seed_demo_data.py` 重新生成。

### Q2: Railway 的 $5 额度够用吗？

**A**: 完全够用。小型 Flask 应用通常只需 $2-3/月。

### Q3: 国内访问速度慢怎么办？

**A**: 
- 演示用途：可接受（通常 2-5 秒加载）
- 正式项目：建议使用国内云服务器

### Q4: 如何更新云端的代码？

**A**: 直接 `git push`，Railway/Render 会自动重新部署。

### Q5: 数据库会丢失吗？

**A**: 不会。PostgreSQL 是持久化的。但建议定期备份。

### Q6: 可以绑定自己的域名吗？

**A**: 可以！Railway 和 Render 都支持自定义域名。

### Q7: 演示模式和开发模式有什么区别？

**A**:

| 特性 | 演示模式 | 开发模式 |
|------|----------|----------|
| 数据库 | SQLite | MySQL |
| 数据 | 预设数据 | 实时数据 |
| 配置 | 简化配置 | 完整功能 |

### Q8: 为什么需要 init_production.py？

**A**: 云平台启动时需要自动初始化数据库结构和默认数据。
