## 🔧 问题修复

### 1. config_demo 导入错误
**问题**: Alt+6 运行时提示 `from config_demo import DemoConfig` 导入失败  
**原因**: config_demo.py 在 scripts 子目录，Python 找不到模块  
**解决**: 将文件移至项目根目录，与 app.py 同级  
**提交**: a4ff446

### 2. APScheduler 重复初始化
**问题**: 测试环境下调度器重复启动导致冲突  
**解决**: 添加全局标志位 `_scheduler_started`，防止重复初始化  
**相关文件**: `app.py`

### 3. 应用初始化条件错误
**问题**: demo 模式下也会自动创建应用实例，导致调度器冲突  
**解决**: 修改条件判断，demo 模式不自动创建实例  
**代码**:
```python
if not os.getenv('TESTING') and not os.getenv('FLASK_ENV') == 'demo':
    app = create_app()
```

## 🚀 部署说明

### Render 部署步骤

1. **连接 GitHub 仓库**
   - 仓库地址: https://github.com/Cguomei/FastApiProject
   - 分支: main

2. **配置 Web Service**
   ```
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn app:create_app() --bind 0.0.0.0:$PORT
   ```

3. **环境变量**
   ```
   FLASK_ENV=demo
   SECRET_KEY=<生成随机字符串>
   ```

4. **数据库**
   - 使用 SQLite (demo.db)
   - 无需配置 PostgreSQL
   - 数据包含在 Git 中

### 本地运行

```bash
# 方式 1: 演示模式
python scripts/seed_demo_data.py
set FLASK_ENV=demo
python app.py

# 方式 2: 开发模式（需要 MySQL）
python init_db.py
python app.py
```

### 访问地址
- 本地: http://localhost:5000
- 演示账户: admin / admin123

## ⚠️ 重要说明

### 密码规则
- **要求**: 至少 6 位
- **字符类型**: 不限（可以是字母、数字、特殊字符的组合）
- **示例**: 
  - ✅ admin123
  - ✅ Test@123!
  - ✅ password
  - ❌ 12345 (不足 6 位)

### 数据库选择
- **演示/学习**: 使用 SQLite (demo.db)
- **生产环境**: 可使用 MySQL 或 PostgreSQL
- **Render 免费层**: 推荐 SQLite（简单稳定）

### 分支策略
- **单一分支**: 只使用 main 分支
- **开发流程**: 直接在 main 上开发和提交
- **避免**: 不要创建额外的功能分支
