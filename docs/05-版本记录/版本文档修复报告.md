## ⚠️ 重要教训

### 版本管理规范

1. **必须先检查现有版本文档**
   
   - 查看 `Test/release_v*.md` 文件
   - 确认最新的版本号
   - 了解版本演进历史

2. **不要随意创建新版本号**
   
   - 遵循语义化版本规范（Semantic Versioning）
   - MAJOR.MINOR.PATCH（主版本。次版本.修订版）
   - 当前最新是 v2.4.0，不是 V2.0

3. **文档版本必须与实际代码对应**
   
   - 检查 Git tags
   - 查看 release notes
   - 确认功能实现情况

### 文档准确性原则

- ✅ 基于事实，不臆测
- ✅ 查证历史记录，不凭空创造
- ✅ 保持版本一致性，所有文档统一
- ✅ 及时更新，反映真实状态
