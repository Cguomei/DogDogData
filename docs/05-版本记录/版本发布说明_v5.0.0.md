#### 修复策略

**全局防护层**（`templates/base.html`）：
```javascript
function escapeHtml(unsafe) {
    if (!unsafe) return unsafe;
    return unsafe
        .replace(/&/g, "&amp;")
        .replace(/</g, "&lt;")
        .replace(/>/g, "&gt;")
        .replace(/"/g, "&quot;")
        .replace(/'/g, "&#039;");
}
```

### 2. 开放重定向漏洞修复

**漏洞详情**：`routes/auth.py` 的 `login` 路由未验证 `next` 参数域名，攻击者可构造 `/auth/login?next=http://evil.com` 跳转到恶意站点。

**修复方式**：添加同源性校验函数 `is_safe_url`。
