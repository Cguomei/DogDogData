## 系统概述

### 设计目标

基于数据新鲜度监控系统，扩展出一套完整的数据分析体系，实现：

- ✅ **数据质量监控**：实时跟踪数据完整性、准确性、新鲜度
- ✅ **价格趋势分析**：按品种统计每日价格变化，识别涨跌趋势
- ✅ **品种热度排行**：基于多维度数据评估品种受欢迎程度
- ✅ **用户行为分析**：记录和分析用户在系统中的行为模式
- ✅ **系统性能监控**：监控API响应时间、错误率等关键指标
- ✅ **每日汇总报告**：自动生成综合分析报告

### 核心价值

1. **决策支持**：为业务决策提供数据支撑
2. **问题发现**：及时发现数据质量问题
3. **趋势预测**：识别价格变化和品种热度趋势
4. **性能优化**：监控系统性能，优化用户体验

## 定时任务配置

### 在app.py中添加定时任务

```python
from apscheduler.schedulers.background import BackgroundScheduler
from utils.analytics_collector import AnalyticsCollector
from datetime import date

def collect_daily_analytics():
    """每日凌晨收集分析数据"""
    try:
        collector = AnalyticsCollector()
        results = collector.collect_all_metrics(date.today())
        
        success_count = sum(1 for v in results.values() if v)
        logger.info(f"每日分析数据收集完成: {success_count}/4 成功")
        
    except Exception as e:
        logger.error(f"每日分析数据收集失败: {str(e)}")

# 初始化调度器
scheduler = BackgroundScheduler()

# 每天凌晨2点执行
scheduler.add_job(
    collect_daily_analytics,
    'cron',
    hour=2,
    minute=0,
    id='daily_analytics_collection'
)

# 启动调度器
scheduler.start()
```

### 定时任务说明

| 任务 | 执行时间 | 说明 |
|------|---------|------|
| 数据质量收集 | 每天02:00 | 统计前一天的数据质量 |
| 价格趋势计算 | 每天02:05 | 计算各品种价格变化 |
| 品种热度统计 | 每天02:10 | 更新品种热度排行 |
| 每日报告生成 | 每天02:15 | 生成综合分析报告 |

## 常见问题

### Q1: 如何查看某个品种的历史价格？

**A**: 使用价格趋势API：


### Q2: 数据收集失败怎么办？

**A**: 检查以下几点：

1. 数据库连接是否正常
2. jd_dogs表是否有数据
3. 查看日志文件：`log/app.log`

```python
# 手动测试数据收集
from utils.analytics_collector import AnalyticsCollector

collector = AnalyticsCollector()
result = collector.collect_data_quality_metrics()
print(f"结果: {result}")
```

---

### Q3: 如何自定义热度评分算法？

**A**: 修改 `analytics_collector.py` 中的 `collect_breed_popularity` 方法：

```python
# 当前算法：基于记录数占比
popularity_score = (total_records / max_count * 100)

# 可以改为综合算法
popularity_score = (
    total_records * 0.4 +
    search_count * 0.3 +
    favorite_count * 0.2 +
    ai_query_count * 0.1
)
```
