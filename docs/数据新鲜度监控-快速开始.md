## 📊 数据状态说明

### 四级状态系统

| 状态 | 颜色 | 阈值 | 说明 | 建议操作 |
|------|------|------|------|----------|
| **fresh** | 🟢 绿色 | < 24小时 | 数据新鲜 | 无需操作 |
| **warning** | 🟡 黄色 | 24-48小时 | 即将过期 | 建议24小时内更新 |
| **stale** | 🔴 红色 | 48-72小时 | 已过期 | 尽快执行爬取 |
| **critical** | ⚫ 深红 | > 72小时 | 严重过期 | 立即更新数据 |

### 品种级监控

每个品种独立计算新鲜度：
- 按最后爬取时间排序
- 显示记录数量和平均价格
- 颜色标识状态
- 支持单独更新特定品种

## 🔧 常见问题

### Q1: 为什么所有品种都显示"999小时前"？

**原因**: 这是旧数据，在添加`crawl_time`字段之前就已经存在。

**解决方案**: 
1. 执行一次新的数据爬取（访问 `/crawler` 页面）
2. 新爬取的数据会有正确的`crawl_time`
3. 或者运行修复脚本：`python scripts/fix_crawl_time.py`

### Q2: 如何更新过时数据？

**方法1 - Web界面**:
1. 访问 http://localhost:5000/crawler
2. 点击"开始爬取"按钮
3. 等待爬取完成

**方法2 - API**:
```bash
curl -X POST "http://localhost:5000/api/crawler/start" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"keywords": ["金毛", "泰迪"]}'
```

### Q3: 如何清理旧数据？

**Web界面**:
- 访问 `/data-freshness` 页面
- 点击"清理旧数据"按钮
- 选择保留天数（默认90天）

**API**:
```bash
curl -X POST "http://localhost:5000/api/data/clean-old" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"days": 90}'
```

**命令行**:
```python
from utils.data_freshness import freshness_checker
deleted = freshness_checker.clean_old_data(days=90)
print(f"删除了 {deleted} 条记录")
```

### Q4: 如何配置自动更新？

目前支持手动触发，自动更新功能可以这样实现：

```python

def auto_update_job():
    """自动更新任务"""
    from utils.data_freshness import auto_scheduler
    
    priority = auto_scheduler.get_update_priority()
    
    if priority['should_update']:
        print("开始自动更新...")
        # 执行爬取逻辑
        # ...

scheduler = BackgroundScheduler()
scheduler.add_job(auto_update_job, 'interval', hours=24)
scheduler.start()
```

## 📝 技术细节

### 数据库字段

```sql
-- jd_dogs表新增字段
ALTER TABLE jd_dogs 
ADD COLUMN crawl_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '爬取时间',
ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间';
```

### SQL查询示例

```sql
-- 检查数据新鲜度
SELECT 
    MAX(crawl_time) as last_crawl,
    COUNT(*) as total_count,
    COUNT(CASE WHEN crawl_time > DATE_SUB(NOW(), INTERVAL 24 HOUR) THEN 1 END) as fresh_count,
    COUNT(CASE WHEN crawl_time BETWEEN DATE_SUB(NOW(), INTERVAL 48 HOUR) AND DATE_SUB(NOW(), INTERVAL 24 HOUR) THEN 1 END) as warning_count,
    COUNT(CASE WHEN crawl_time < DATE_SUB(NOW(), INTERVAL 48 HOUR) THEN 1 END) as stale_count
FROM jd_dogs;

-- 获取各品种新鲜度
SELECT 
    dog_name,
    COUNT(*) as count,
    MAX(crawl_time) as last_crawl,
    AVG(price) as avg_price,
    TIMESTAMPDIFF(HOUR, MAX(crawl_time), NOW()) as hours_ago
FROM jd_dogs
GROUP BY dog_name
ORDER BY hours_ago DESC
LIMIT 20;
```
