# 京东狗狗数据爬虫 - 使用说明

## 📋 功能概述

本爬虫用于从京东商城自动爬取狗狗价格数据，并保存到数据库中。支持：
- ✅ 关键词搜索爬取
- ✅ 智能去重（根据品种+店铺）
- ✅ 增量更新（只更新价格和评论数）
- ✅ 防封策略（随机延迟、User-Agent轮换）
- ✅ 数据统计报告

## 🔧 高级用法

### 1. 自定义品种爬取

```python
crawler = JDDogCrawler()

# 爬取特定品种
dogs = crawler.search_dogs(keyword="泰迪犬", max_pages=5)
dogs = crawler.search_dogs(keyword="金毛寻回犬", max_pages=3)
```

### 2. 批量爬取多个品种

```python
keywords = ["泰迪", "金毛", "哈士奇", "柯基", "拉布拉多"]

for keyword in keywords:
    print(f"正在爬取: {keyword}")
    dogs = crawler.search_dogs(keyword=keyword, max_pages=2)
    crawler.save_to_database(dogs)
    time.sleep(5)  # 每个品种之间等待5秒
```

### 3. 定时任务（需要配置APScheduler）

在 `app.py` 中添加：

```python
@scheduler.task('interval', id='daily_crawl', hours=24)
def daily_crawl():
    """每天凌晨2点自动爬取"""
    with app.app_context():
        crawler = JDDogCrawler()
        dogs = crawler.search_dogs(keyword="宠物狗", max_pages=3)
        crawler.save_to_database(dogs)
```

## ⚠️ 注意事项

### 1. 遵守robots.txt

- 请遵守京东的robots.txt协议
- 控制爬取频率，避免对服务器造成压力
- 仅用于个人学习和研究

### 2. 反爬策略

当前实现的反爬措施：
- ✅ User-Agent伪装
- ✅ 随机延迟（2-5秒）
- ✅ Session保持

如果遇到封禁：
- 增加延迟时间（改为5-10秒）
- 减少单次爬取页数
- 使用代理IP池（需自行实现）

### 3. 数据质量

- 价格可能因促销活动波动
- 部分商品可能无价格信息
- 建议定期更新数据

## 🐛 常见问题

### Q1: 爬取速度慢？

**A:** 这是正常的，为了避免被封，设置了2-5秒的随机延迟。如果需要更快：
- 减少页数
- 降低延迟（不推荐）

### Q2: 爬取失败率高？

**A:** 可能原因：
- 网络连接问题
- 京东反爬限制
- HTML结构变化

解决方法：
- 检查网络连接
- 增加延迟时间
- 查看日志文件 `logs/crawler.log`

### Q3: 数据重复？

**A:** 爬虫会自动去重（根据品种+店铺）。如果仍有重复：
- 检查数据库唯一索引
- 手动清理重复数据

```sql
DELETE t1 FROM jd_dogs t1
INNER JOIN jd_dogs t2 
WHERE t1.id > t2.id 
AND t1.dog_name = t2.dog_name 
AND t1.shop_name = t2.shop_name;
```

### Q4: 价格为0？

**A:** 可能原因：
- 商品已售罄
- 价格需要登录才能查看
- HTML解析失败

解决方法：
- 过滤掉价格为0的数据
- 手动补充价格信息

## 📈 性能优化建议

### 1. 并发爬取（高级）

```python
from concurrent.futures import ThreadPoolExecutor

def crawl_keyword(keyword):
    crawler = JDDogCrawler()
    dogs = crawler.search_dogs(keyword=keyword, max_pages=2)
    return crawler.save_to_database(dogs)

keywords = ["泰迪", "金毛", "哈士奇"]

with ThreadPoolExecutor(max_workers=3) as executor:
    results = executor.map(crawl_keyword, keywords)
```

**注意：** 并发会增加被封风险，谨慎使用。

### 2. 增量更新

只爬取最近7天的新商品：

```python
# 在parse_search_page中添加时间过滤
if item_date < (datetime.now() - timedelta(days=7)):
    continue
```

### 3. 缓存机制

缓存已爬取的URL，避免重复爬取：

```python
self.crawled_urls = set()

if url in self.crawled_urls:
    return []
self.crawled_urls.add(url)
```
