## 系统概述

### 背景

在狗狗数据分析系统中，数据时效性至关重要。过时的数据会导致：
- 价格分析不准确
- 品种趋势判断错误
- 用户决策失误

### 解决方案

开发数据新鲜度监控系统，实现：
- ✅ 实时监控数据时效性
- ✅ 四级状态预警机制
- ✅ 品种级细粒度监控
- ✅ 智能更新建议
- ✅ 一键清理旧数据

### 核心价值

1. **数据质量保障**：确保分析基于最新数据
2. **自动化运维**：减少人工检查成本
3. **可视化监控**：直观展示数据状态
4. **智能决策支持**：提供更新优先级建议

## 核心功能

### 1. 数据新鲜度检查器 (DataFreshnessChecker)

#### 功能说明

检查整体和各品种的数据新鲜度，返回状态评估和建议。

#### 四级状态系统

| 状态 | 颜色 | 阈值 | 含义 | 建议操作 |
|------|------|------|------|----------|
| **fresh** | 🟢 绿色 | < 24小时 | 数据新鲜 | 无需操作 |
| **warning** | 🟡 黄色 | 24-48小时 | 即将过期 | 24小时内更新 |
| **stale** | 🔴 红色 | 48-72小时 | 已过期 | 尽快执行爬取 |
| **critical** | ⚫ 深红 | > 72小时 | 严重过期 | 立即更新数据 |

#### 核心方法

##### check_data_freshness()

检查整体数据新鲜度。

**返回值结构**:
```python
{
    'status': 'fresh',  # fresh/warning/stale/critical/no_data
    'message': '数据新鲜（0.0小时前更新）',
    'recommendation': '数据状态良好，无需操作',
    'total_records': 2278,
    'fresh_records': 2278,
    'warning_records': 0,
    'stale_records': 0,
    'freshness_percentage': 100.0,
    'hours_since_last_crawl': 0.0
}
```

**SQL查询**:
```sql
SELECT 
    MAX(crawl_time) as last_crawl,
    MAX(updated_at) as last_update,
    COUNT(*) as total_count,
    COUNT(CASE WHEN crawl_time > DATE_SUB(NOW(), INTERVAL 24 HOUR) THEN 1 END) as fresh_count,
    COUNT(CASE WHEN crawl_time BETWEEN DATE_SUB(NOW(), INTERVAL 48 HOUR) AND DATE_SUB(NOW(), INTERVAL 24 HOUR) THEN 1 END) as warning_count,
    COUNT(CASE WHEN crawl_time < DATE_SUB(NOW(), INTERVAL 48 HOUR) THEN 1 END) as stale_count
FROM jd_dogs;
```

##### get_breed_freshness()

获取各品种数据新鲜度列表。

**返回值结构**:
```python
[
    {
        'breed': '金毛',
        'count': 250,
        'avg_price': 4468.01,
        'last_crawl': datetime(2026, 5, 12, 19, 14, 43),
        'hours_ago': 0.0,
        'status': 'fresh'
    },
    ...
]
```

**SQL查询**:
```sql
SELECT 
    dog_name,
    COUNT(*) as count,
    MAX(crawl_time) as last_crawl,
    AVG(price) as avg_price,
    TIMESTAMPDIFF(HOUR, MAX(crawl_time), NOW()) as hours_ago
FROM jd_dogs
GROUP BY dog_name
ORDER BY hours_ago DESC
LIMIT 20;
```

##### clean_old_data(days=90)

清理指定天数前的旧数据。

**参数**:
- `days`: 保留天数，默认90天

**返回值**:
- `int`: 删除的记录数

**SQL查询**:
```sql
DELETE FROM jd_dogs 
WHERE crawl_time < DATE_SUB(NOW(), INTERVAL :days DAY);
```

### 2. 自动更新调度器 (AutoUpdateScheduler)

#### 功能说明

智能评估数据更新优先级，提供自动化更新建议。

#### 核心方法

##### should_auto_update()

判断是否应该自动更新。

**返回值**: `bool`

**判断逻辑**:
```python
if status == 'critical':
    return True
elif status == 'stale':
    return True
elif status == 'warning':
    return False  # 仅警告，不自动更新
else:
    return False
```

##### get_update_priority()

获取详细的更新优先级信息。

**返回值结构**:
```python
{
    'priority': {
        'level': 'none',  # none/low/medium/high/critical
        'urgency': 'not_needed',  # not_needed/suggested/urgent/immediate
        'recommended_action': '无需操作'
    },
    'should_update': False,
    'reason': '数据状态良好'
}
```

**优先级映射**:

| 数据状态 | 优先级等级 | 紧急程度 | 推荐操作 |
|---------|-----------|---------|---------|
| fresh | none | not_needed | 无需操作 |
| warning | low | suggested | 建议更新 |
| stale | high | urgent | 尽快更新 |
| critical | critical | immediate | 立即更新 |

## 数据库设计

### jd_dogs表结构

```sql
CREATE TABLE jd_dogs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    dog_name VARCHAR(100) NOT NULL COMMENT '品种名称',
    price DECIMAL(10, 2) COMMENT '价格',
    shop_name VARCHAR(200) COMMENT '店铺名称',
    location VARCHAR(100) COMMENT '地区',
    age VARCHAR(50) COMMENT '年龄',
    gender VARCHAR(10) COMMENT '性别',
    weight VARCHAR(50) COMMENT '体重',
    level VARCHAR(50) COMMENT '级别',
    image_url TEXT COMMENT '图片URL',
    product_url TEXT COMMENT '商品链接',
    
    -- 新增字段
    crawl_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '爬取时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    
    INDEX idx_dog_name (dog_name),
    INDEX idx_crawl_time (crawl_time),
    INDEX idx_updated_at (updated_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='京东狗狗数据表';
```

### 索引优化

```sql
-- 复合索引：按品种和爬取时间查询
CREATE INDEX idx_breed_crawl ON jd_dogs (dog_name, crawl_time);

-- 覆盖索引：新鲜度统计查询
CREATE INDEX idx_crawl_stats ON jd_dogs (crawl_time, dog_name, price);
```

### 迁移脚本

```python

    db.session.execute(text("""
        ALTER TABLE jd_dogs 
        ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP 
        COMMENT '更新时间'
```

## 故障排除

### 常见问题

#### 1. Unknown column 'crawl_time'

**症状**:
```
OperationalError: (1054, "Unknown column 'crawl_time' in 'field list'")
```

**原因**: 数据库未执行迁移脚本

**解决方案**:
```bash
python scripts/migrate_add_crawl_fields.py
```

#### 2. SchedulerAlreadyRunningError

**症状**:
```
apscheduler.schedulers.SchedulerAlreadyRunningError
```

**原因**: 多次调用create_app导致scheduler重复初始化

**解决方案**:
在脚本中使用简化的Flask应用创建：
```python
from flask import Flask
from config import DevelopmentConfig

app = Flask(__name__)
app.config.from_object(DevelopmentConfig)
db.init_app(app)
```

#### 3. 所有品种显示"999小时前"

**原因**: 旧数据没有crawl_time，迁移时使用了默认值

**解决方案**:
1. 执行一次新爬取
2. 或运行修复脚本：
   ```bash
   python scripts/fix_crawl_time.py
   ```

#### 4. API返回401 Unauthorized

**原因**: 未登录或token过期

**解决方案**:
1. 先登录获取token
2. 在请求头中添加：`Authorization: Bearer YOUR_TOKEN`
