# AI助手 - 自动日志系统说明

## ✅ 已实现的日志功能

### 1. 完整的日志记录

所有AI助手的操作都会自动记录到 `log/ai_assistant.log` 文件中，包括：

#### 📝 记录的详细信息

**请求级别日志**：
- ✅ 每个请求的唯一ID（时间戳精确到微秒）
- ✅ 用户信息（用户名、用户ID）
- ✅ 请求数据（JSON格式）
- ✅ 问题类型分类结果
- ✅ 处理参数

**业务逻辑日志**：
- ✅ 价格查询：品种、查询结果、平均价格
- ✅ 品种查询：品种名、数据库结果
- ✅ 推荐请求：处理流程
- ✅ 对比请求：对比的品种列表
- ✅ 通用问答：处理标记

**模型调用日志**：
- ✅ 模型类型和名称
- ✅ API地址
- ✅ 请求payload（前500字符）
- ✅ 响应状态码
- ✅ 响应内容（前500字符）
- ✅ 回复长度

**错误日志**：
- ✅ 完整堆栈跟踪（traceback）
- ✅ 错误发生时的请求数据
- ✅ 用户信息
- ✅ 错误类型和详细描述

### 3. 日志查看工具

#### 方式1：Web界面（推荐）

访问：`http://localhost:5000/ai-logs`（需要管理员权限）

**功能**：
- 📊 实时统计（总行数、错误数、警告数、文件大小）
- 🔍 过滤显示（全部/错误/警告/信息）
- ⚠️ 快速查看错误（带堆栈跟踪）
- 🔄 刷新日志
- 🗑️ 清空日志

**界面特点**：
- 深色主题，适合长时间查看
- 颜色区分不同级别（INFO蓝色、WARNING黄色、ERROR红色）
- 错误堆栈跟踪折叠显示
- 最新日志在前

#### 方式2：直接查看日志文件

```bash
# Windows PowerShell
Get-Content log/ai_assistant.log -Tail 50 -Wait

# 或使用记事本打开
notepad log/ai_assistant.log
```

#### 方式3：API接口

```bash
# 获取最近100条日志
curl http://localhost:5000/api/ai/logs/recent?lines=100

# 只获取错误日志
curl http://localhost:5000/api/ai/logs/errors

# 清空日志
curl -X POST http://localhost:5000/api/ai/logs/clear
```

### 4. 日志文件位置

```
D:\PycharmProjects\fastApiProject\log\ai_assistant.log
```

**注意**：

- 不会自动轮转（需要手动清理）
- 使用UTF-8编码，支持中文
- 建议定期清理（通过Web界面或API）

## 🔧 日志级别说明

| 级别 | 说明 | 何时使用 |
|------|------|---------|
| DEBUG | 调试信息 | 详细的请求/响应数据 |
| INFO | 一般信息 | 正常业务流程 |
| WARNING | 警告信息 | 非致命问题（如未找到数据） |
| ERROR | 错误信息 | 异常、失败、堆栈跟踪 |

## 📊 日志包含的关键信息

### 每次请求都有唯一ID

格式：`YYYYMMDDHHmmssffffff`（年月日时分秒微秒）

例如：`20260509173524123456`

**作用**：
- 追踪完整的请求链路
- 关联相关日志条目
- 便于问题定位

### 用户追踪

每条日志都包含：
- 用户名
- 用户ID（如果可用）

**作用**：
- 了解哪个用户触发了问题
- 分析用户使用模式
- 安全审计

### 性能监控

通过日志可以分析：
- 请求处理时间（通过时间戳计算）
- 模型响应时间
- 数据库查询频率
- 错误率

## 🎯 实际使用场景

### 场景1：用户报告"AI回复错误"

**排查步骤**：
1. 访问 `/ai-logs` 查看最近的错误
2. 找到对应的请求ID
3. 查看完整的请求数据和堆栈跟踪
4. 定位问题原因

### 场景2：模型响应很慢

**排查步骤**：
1. 查看日志中的时间戳
2. 计算"调用本地模型"到"回复成功"的时间差
3. 检查是否有超时错误

### 场景3：某个功能不工作

**排查步骤**：
1. 让用户描述操作
2. 根据时间和用户名查找对应日志
3. 查看问题分类是否正确
4. 检查数据库查询结果
5. 查看模型调用是否成功

## 💡 最佳实践

### 1. 定期检查错误

每天访问一次 `/ai-logs`，查看是否有新的错误。

### 2. 监控警告信息

警告可能预示着潜在问题，及时处理。

### 3. 保留重要日志

在清空日志前，如果有重要错误，先备份：
```bash
copy log\ai_assistant.log log\ai_assistant_backup_20260509.log
```

### 4. 分析用户行为

通过日志了解：
- 用户最常问什么问题
- 哪些功能使用频率高
- 哪些问题容易出错

## 🔒 安全性

### 权限控制

- 日志查看页面：仅管理员可访问
- 日志API：仅登录用户可访问
- 清空日志：仅管理员可操作

### 敏感信息保护

日志中**不会**记录：
- ❌ 用户密码
- ❌ API密钥
- ❌ 数据库密码
- ❌ 完整的请求体（超过500字符会截断）
