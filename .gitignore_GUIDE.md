# Git 忽略规则说明

## 📋 已添加到 .gitignore 的内容

### AI 助手配置（Lingma）
```
.lingma/
```

**原因**：
- `.lingma/` 目录包含 AI 助手的规则和 Skills 配置
- 这些是开发者的个人配置，与项目代码无关
- 不同开发者可能有不同的 AI 助手配置
- 避免将个人配置污染到项目仓库

---

## 🔍 其他重要的忽略规则

### 1. 敏感信息
```
.env
.env.local
.env.production
```
- 包含数据库密码、API密钥等敏感信息
- **严禁上传到公开仓库**

### 2. Python 缓存
```
__pycache__/
*.pyc
*.pyo
*.pyd
```
- Python 编译后的字节码文件
- 自动生成，无需版本控制

### 3. 虚拟环境
```
venv/
env/
ENV/
```
- 本地 Python 虚拟环境
- 体积大，可通过 requirements.txt 重建

### 4. IDE 配置
```
.idea/
.vscode/
*.swp
*.swo
```
- 个人编辑器配置
- 不同开发者使用不同 IDE

### 5. 日志文件
```
*.log
log/
```
- 运行时生成的日志
- 体积大，内容动态变化

### 6. 内部测试文档
```
Test/*.txt
Test/reports/
test_pet_report/
Test/docs/
Test/test_tools/
```
- 内部测试报告和分析文档
- 保密内容，不对外公开

### 7. 临时脚本
```
scripts/
```
- 一次性使用的工具脚本
- 不属于核心项目代码

### 8. Playwright 相关文件
```
node_modules/
/test-results/
/playwright-report/
/blob-report/
/playwright/.cache/
/playwright/.auth/
```
- UI 测试生成的文件和缓存
- 体积大，可重新生成

## ⚠️ 常见错误

### ❌ 错误做法

1. **提交敏感信息**
   ```bash
   git add .env  # 绝对禁止！
   ```

2. **提交大型二进制文件**
   ```bash
   git add *.db  # SQLite 数据库文件
   git add node_modules/  # 依赖包
   ```

3. **提交个人配置**
   ```bash
   git add .lingma/  # AI 助手配置
   git add .idea/  # IDE 配置
   ```

4. **提交流水文件**
   ```bash
   git add *.log  # 日志文件
   git add Test/reports/  # 测试报告
   ```

### ✅ 正确做法

1. **只提交必要的代码和文档**
   ```bash
   git add routes/ models.py README.md
   ```

2. **使用 .gitignore 排除不必要的文件**
   ```bash
   # 在 .gitignore 中添加
   .env
   __pycache__/
   .lingma/
   ```

3. **提交前检查**
   ```bash
   git status  # 查看将要提交的文件
   git diff --cached  # 查看具体变更
   ```

## 🛠️ 如何修复已提交的敏感文件

如果不小心提交了敏感文件，需要：

### 1. 从 Git 历史中移除
```bash
# 从当前 commit 移除
git rm --cached .env

# 从所有历史中移除（谨慎使用）
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch .env' \
  --prune-empty --tag-name-filter cat -- --all
```

### 2. 添加到 .gitignore
```bash
echo ".env" >> .gitignore
git add .gitignore
git commit -m "Add .env to gitignore"
```

### 3. 强制推送（会改写历史）
```bash
git push origin main --force
```

**注意**：如果已经推送到远程，需要通知所有协作者重新克隆仓库。

## 📝 最佳实践

### 1. 提交前检查清单
- [ ] 没有包含 `.env` 等敏感文件
- [ ] 没有包含 `__pycache__/` 等缓存文件
- [ ] 没有包含 `.lingma/` 等个人配置
- [ ] 没有包含大型二进制文件
- [ ] Commit message 清晰明了

### 2. 定期清理
```bash
# 查看未跟踪的文件
git status

# 清理未跟踪的文件（谨慎使用）
git clean -fdn  # 先预览
git clean -fd   # 再执行
```
