## 📹 演示流程

### 场景1: 首次使用（30秒）

```
1. 双击 Test\start_test_console.bat
   ↓
2. 等待浏览器自动打开 http://localhost:5555
   ↓
3. 看到漂亮的仪表盘界面
   ↓

### 场景2: 日常测试（1分钟）

```
1. 启动控制台（如果已运行可跳过）
   ↓
2. 选择测试类型：
   - API测试 → 快速验证后端

   ↓

## 💡 使用技巧

### 技巧1: 快速定位问题

```
步骤:
1. 观察统计卡片中的"失败数量"
2. 如果 > 0，查看终端输出的红色行
3. 记录失败的测试名称
4. 单独运行该模块测试
5. 查看详细错误信息
```

### 技巧2: 提高测试效率

```
建议:
- 开发中: 只运行相关模块测试
- 提交前: 运行API测试
- 每日: 运行全面测试
- 每周: 生成覆盖率报告
```

### 技巧3: 日志管理

```
最佳实践:
- 每次重要测试后导出日志
- 按日期命名: test_log_20260510.txt
- 保留最近的7天日志
- 失败的测试日志永久保存
```

## 🆘 故障排除演示

### 问题1: 服务无法启动

**现象**: 
- 命令行报错
- 端口被占用

**解决**:
```bash
# 检查端口占用
netstat -ano | findstr :5555

# 结束占用进程
taskkill /F /PID <进程ID>

# 重新启动
python Test/test_server.py
```

---

### 问题2: 按钮点击无反应

**现象**:
- 点击按钮没任何反应
- 终端没有输出

**排查**:
1. 按F12打开浏览器开发者工具
2. 查看Console标签页的错误
3. 查看Network标签页的请求状态
4. 检查后端服务是否运行

**解决**:
- 刷新页面
- 重启后端服务
- 检查网络连接

---

### 问题3: 测试输出不更新

**现象**:
- 测试已开始但终端空白
- 进度条不动

**原因**:
- 网络问题导致轮询失败
- 后端服务异常

**解决**:
1. 刷新页面
2. 检查后端服务日志
3. 查看浏览器控制台错误
4. 重启服务
