# .lingma 配置目录

本目录包含 Lingma AI 助手的规则和 Skills 配置。

## 📁 目录结构

```
.lingma/
├── rules/              # 项目规则目录
│   ├── project-management.md    # 项目管理规范
│   └── SKILLS_GUIDE.md          # Skills 使用指南
├── skills/             # Skills 目录
│   └── tdd-development.md       # TDD 开发工作流程
└── skills-main/        # 官方 Skills（不要修改）
```

## 📋 规则文件（rules/）

规则文件定义了 AI 助手在项目中必须遵循的规范和原则。

### 当前规则

- **project-management.md**: 项目管理核心规范
  - 测试驱动开发（TDD）
  - 渐进式开发策略
  - 代码质量管理
  - 工作流程
  - 禁止行为和必须执行的事项

### 如何让规则生效

1. **规则会自动加载**：Lingma 会自动读取 `.lingma/rules/` 目录下的所有 `.md` 文件
2. **规则优先级高**：项目级规则会覆盖通用规则
3. **定期更新**：根据项目需要随时添加或修改规则

### 添加新规则

创建新的 `.md` 文件到 `rules/` 目录：

```markdown
# 规则名称

## 规则内容

详细说明规则的要点和要求。

## 示例

提供具体的使用示例。
```

## 🎯 Skills 文件（skills/）

Skills 是 AI 助手的技能包，教导 AI 如何执行特定任务。

### 当前 Skills

- **tdd-development.md**: 测试驱动开发工作流程
  - 红-绿-重构循环
  - 详细的执行步骤
  - 示例对话
  - 最佳实践

### 如何使用 Skills

#### 方法一：自动应用

当您的需求匹配 Skill 的描述时，AI 会自动应用相应的技能。

例如：
- 你说"添加一个新功能" → AI 可能应用 TDD Skill
- 你说"修复这个 bug" → AI 可能应用 TDD Skill

#### 方法二：手动指定

在对话中明确指定使用某个 Skill：

```
请使用 tdd-development skill 来实现这个功能
```

或者：

```
按照 TDD 流程来做
```

#### 方法三：使用命令

```
/create-skill
```

然后按照提示创建新的 Skill。

### 创建新 Skill

1. 在 `skills/` 目录创建新的 `.md` 文件
2. 添加 frontmatter（元数据）：
   ```markdown
   ---
   name: skill-name
   description: 技能描述
   version: 1.0.0
   tags: [tag1, tag2]
   ---
   ```
3. 编写技能内容：
   - 何时使用
   - 执行步骤
   - 示例
   - 注意事项

### Skill 最佳实践

✅ **好的 Skill 特征**：
- 目标明确，职责单一
- 步骤清晰，可执行
- 有具体示例
- 包含常见问题的解决方案

❌ **避免的问题**：
- 过于宽泛，缺乏针对性
- 步骤模糊，难以执行
- 没有实际示例
- 与现有 Skill 重复

## 💡 为什么您的规则之前没有被执行？

可能的原因和解决方案：

### 原因 1：规则位置不正确

❌ **错误**：规则放在其他地方  
✅ **正确**：规则必须放在 `.lingma/rules/` 目录

### 原因 2：规则格式不规范

❌ **错误**：随意的文本描述  
✅ **正确**：结构化的 Markdown 格式，清晰的标题和列表

### 原因 3：规则不够具体

❌ **错误**："要写好代码"  
✅ **正确**："新功能必须先写测试，测试覆盖率不低于 80%"

### 原因 4：没有在对话中引用规则

❌ **错误**：期望 AI 自动记住所有规则  
✅ **正确**：在关键节点提醒 AI 遵循规则，或使用 Skills

### 原因 5：规则冲突

❌ **错误**：多条规则相互矛盾  
✅ **正确**：确保规则之间的一致性

## 🚀 提高规则执行率的技巧

### 1. 在对话开始时明确规则

```
请遵循项目的项目管理规范（.lingma/rules/project-management.md）
```

### 2. 使用 Skills 而不是口头规则

Skills 比口头规则更有效，因为：
- 结构化更好
- 有具体示例
- AI 更容易理解和执行

### 3. 分阶段引导

不要一次性要求太多，分步骤引导：

```
第一步：先写测试用例
第二步：实现功能代码
第三步：运行测试验证
```

### 4. 及时反馈

如果 AI 没有遵循规则，及时指出：

```
请注意，按照项目规范，应该先写测试再实现功能
```

### 5. 使用示例对话

在规则文件中提供示例对话，让 AI 学习正确的交互方式。

## 📊 规则执行检查清单

在与 AI 交互时，可以检查：

- [ ] AI 是否先确认了需求理解？
- [ ] AI 是否提供了多个方案供选择？
- [ ] AI 是否遵循了 TDD 流程（如适用）？
- [ ] AI 是否在完成后运行了测试？
- [ ] AI 是否总结了所做的更改？

如果没有，可以提醒 AI 查看相关规则。
